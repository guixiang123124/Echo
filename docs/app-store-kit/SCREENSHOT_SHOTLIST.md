# 截图拍摄脚本（iOS + macOS）

## 通用拍摄规则
- 使用真实功能界面，不用占位图。
- 所有截图语言与元数据语言一致（中文一套、英文一套）。
- 每张图只表达 1 个卖点。
- 先录制稳定流程，再截关键帧。

## iOS 截图脚本（建议 6 张）
1. 首页主界面
- 画面：主按钮、核心入口
- 标题文案：`说话，几秒后自动成文`

2. 录音中
- 画面：录音状态 + 波形/计时
- 标题文案：`一键开始，一键结束`

3. 转写结果
- 画面：转写文本结果
- 标题文案：`语音快速转为文字`

4. 自动插入场景
- 画面：目标输入框中成功写入文字
- 标题文案：`自动写入当前输入框`

5. 历史记录
- 画面：历史列表 + 一条清晰记录
- 标题文案：`本地历史，可追溯`

6. 设置页
- 画面：模型、语言、权限配置
- 标题文案：`按你的习惯配置`

## macOS 截图脚本（建议 6 张）
1. 菜单栏 Ready 状态
- 标题文案：`随时可说，快速输入`

2. 开始录音状态
- 标题文案：`触发即录，结束即转写`

3. 转写后自动插入
- 画面：任意输入框中看到写入结果
- 标题文案：`无需手动粘贴`

4. 权限页
- 画面：Microphone / Input / Accessibility 已授权
- 标题文案：`授权一次，长期使用`

5. 历史与回放
- 标题文案：`记录可查，可回放`

6. 诊断信息
- 画面：Hotkey Monitor On、最近日志
- 标题文案：`状态可见，问题可追踪`

## 拍摄执行顺序（建议）
1. 先拍“正常主流程”相关图（1/2/3/4）
2. 再拍“设置与管理”相关图（5/6）
3. 最后统一修文案和裁剪

## 出图命名规范
- iOS: `ios_01_home.png` ... `ios_06_settings.png`
- macOS: `mac_01_ready.png` ... `mac_06_diagnostics.png`

## 最终提交前检查
- 每张图文案无错别字
- 截图内容与真实功能一致
- 无测试账号、API Key、个人隐私信息泄漏

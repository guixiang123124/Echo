# Echo 项目总结与后续计划（2026-02-17）

> 这份文档只聚焦项目本身：已完成能力、当前状态、关键问题、下一阶段计划。

---

## 1) 项目愿景（当前共识）

Echo 的目标是做一个可在 **macOS + iOS** 使用的语音输入/转写产品，核心体验是：

- 录音与转写稳定、低延迟
- 多 ASR Provider 可切换并可比较（OpenAI / Volcano / Ark / Deepgram 等）
- 自动纠错（Auto Edit）可控、可解释、可追踪
- 历史记录可同步与可管理
- 账号与登录体系稳定（为后续云同步与订阅能力打基础）

---

## 2) 已有成果（项目层面）

### 2.1 基础产品能力

- iOS + macOS 双端工程可运行
- OpenAI ASR 路径可跑（多模型对比能力已具备）
- 录音历史存储与展示链路可用
- Provider 选择与设置界面已建立（含 Volcano/Ark 等配置入口）

### 2.2 自动纠错（Auto Edit）能力增强

- 新增 History/Recording History 的 debug indicator：
  - 显示 Auto Edit provider
  - 显示 edited / no change
- 修正 `correctionProviderId` 写入语义：
  - 仅在真正执行纠错时写入 provider id
  - 避免“看起来启用但实际未运行”的误判

### 2.3 ASR Benchmark 体系

- 已有 ASR benchmark CLI 与报告输出机制
- 对给定两段音频可输出多模型 latency/transcript 报告
- 报告目录：`Packages/EchoCore/reports/asr/`

## 4) 这段时间中有价值的“思维火花”

1. **“可观测性优先”**：
   对于 ASR/Auto Edit，必须在历史里可见 provider 与状态，避免黑盒。

2. **“多模型不是配置列表，而是对比体系”**：
   只有统一输入、统一指标、统一报告，模型切换才有产品意义。

3. **“过程不等于交付”**：
   调试很多不代表任务完成，必须沉淀成可复现结果与证据。

4. **“账号体系是后续商业能力的地基”**：
   登录稳定性直接影响云同步、订阅与长期留存。

## 5) 下一阶段推进计划（建议按优先级）

1. **Volcano 跑通并进入 A/B 报告**
